# 설비 수주 자동응답 매크로 - 개발정의서 (macro_v1_2 기준)

## 1. 프로젝트 개요

| 항목 | 내용 |
|---|---|
| 프로젝트명 | 설비 수주 자동응답 매크로 |
| 현재 버전 | macro_v1_2 GUI |
| 실행 파일 | `macro_v1_2_GUI.py` |
| 개발 언어 | Python 3.x |
| 대상 OS | Windows 10/11 |
| 대상 앱 | 카카오톡 PC (오픈채팅방) |

## 2. 핵심 동작 원리

```
카카오톡 PC 채팅창 캡처 → OCR 텍스트 추출 → 조건 매칭 → 자동 '네' 전송
```

### 동작 흐름
1. 사용자가 감시할 **지역**, **채팅방 이름**, **응답 문구**를 설정
2. 시작 버튼 클릭 → 카카오톡 채팅창을 주기적으로 캡처 (기본 0.3초)
3. Tesseract OCR로 채팅 영역 이미지에서 텍스트 추출
4. 추출된 텍스트에서 **지역 + 작업 + 수수료** 3가지 조건 파싱
5. 조건 충족 시 해당 채팅창에 응답 문구 자동 전송
6. 동일 조건 메시지 중복 전송 방지

## 3. 인식 조건 (3가지 모두 충족 시 응답)

### 3-1. 지역
- 사용자가 UI에서 **시/도 → 구/군/시 → 동** 단위로 선택
- `(전체)` 선택 시 해당 구/군/시의 모든 동을 감시
- 내장 지역 데이터: 서울(7구), 경기(광명/안산/안양/군포), 인천(10구/군)

### 3-2. 작업 종류
| 정규화 이름 | 인식 변형 |
|---|---|
| 싱막 (싱크대막힘) | 싱막, 성막, 생막, 씽막 |
| 하막 (하수구막힘) | 하막, 하 막 |
| 역류 | 역류 |
| 누수 | 누수 |
| 싱 크 대 막힘 | 싱크대막힘 |
| 하 수 구 막힘 | 하수구막힘 |

### 3-3. 수수료

**조건:**
- 1 ~ 30 범위 숫자

**허용 패턴:**
- `수수10`
- `수수 10`
- `수수료10`
- `수수료 10%`
- 숫자 단독 (예: `10`)
- OCR 공백 (`수 수 료 10`) 처리

**제외:**
- `0`
- `31` 이상
- 숫자 없음
- `%` 포함 여부 무관

### 인식 예시
```
"구로 싱막 수수15"      → 지역: 구로동, 작업: 싱막, 수수료: 15
"대방동 하막 수수료10%"  → 지역: 대방동, 작업: 하막, 수수료: 10
"관양 싱 막 수 수 10"   → 지역: 관양동, 작업: 싱막, 수수료: 10
```

## 4. OCR 보정 로직

### 4-1. 텍스트 정규화
- 공백 제거
- OCR 분절 텍스트 결합

### 4-2. 작업명 보정
- 띄어쓰기 분리 자동 결합
- 유사 문자 변환

### 4-3. 지역 보정
- `X등` → `X동` (대방등→대방동, 상도등→상도동 등 범용)
- `금전` → `금천`
- `가포` → `마포`
- `과악` → `관악`
- suffix 제거
- 부분 문자열 매칭

### 4-4. 지역 매칭 전략

| 모드 | 조건 | 동작 |
|---|---|---|
| **일반 모드** | 감시 지역 20개 미만 | 정확 매칭 + fuzzy(1글자 차이 허용) |
| **Strict 모드** | 감시 지역 20개 이상 (`(전체)` 포함 시) | 정확 매칭만, fuzzy 비활성 |

**일반 모드 매칭 규칙:**
1. **정확 매칭**: 선택 지역의 variants (동/구/시 suffix 조합)가 텍스트에 포함
2. **Fuzzy 매칭**: normalize 결과가 **3글자 이상**이고, 1글자만 다른 경우 허용 (macro_v1_2 강화)
   - 예: `노량잔` → `노량진` (diff=1, 3글자) ✓
   - 예: `장화` → `강화` (normalize 2글자) ✗ **차단** — 2글자 fuzzy는 오탐률이 높아 비활성

**Strict 모드 추가 규칙 (macro_v1_0~v1_1 적용):**
- Fuzzy 매칭 완전 비활성
- 2글자 지역명 (예: `구로`, `당산`)은 메시지 맨 앞에 위치할 때만 허용
- 3글자 이상 지역명은 일반 substring 매칭 유지
- `(전체)` 또는 광범위 지역 선택 시 자동 적용

## 5. 후보 처리 로직

### Strong 후보
- 지역명과 수수료가 compact 텍스트에서 또렷하게 인식된 경우
- 일반 모드: 1회 확인 후 즉시 전송
- Strict 모드 (macro_v1_0~v1_1): 오탐 방지를 위해 최소 2회 확인
- 예: `구로싱막수수15` → (일반) 즉시, (strict) 2회 확인 후 전송

### Weak 후보
- OCR이 불안정하게 읽힌 경우
- 일반 모드: 2회 확인 후 전송
- Strict 모드 (macro_v1_0~v1_1): 3회 확인 후 전송

### 수수료 애매 후보 (macro_v1_1 강화)
- `수수1` 처럼 OCR이 수수료 1자리만 읽은 경우
- `%` 또는 `료` 없이 숫자 1만 단독 등장 시 → 최소 3회 확인 필수

### Multi-pending 구조
- 여러 후보를 동시에 추적 (pending_candidates dict)
- 한 루프당 1개만 전송 (카카오톡 입력 안정성)
- 전송 간격 (기본 2.0초) 준수
- OCR이 1~2회 튀어도 후보 즉시 삭제하지 않음 (PENDING_TTL = 1.6초)

## 6. 중복 방지

- `지역|작업|수수료` 조합으로 고유 키 생성
- 동일 키는 1회만 전송 (processed_keys)
- 기준선 설정: 감시 시작 시 화면에 이미 있는 메시지는 전체 무시 등록

### 수수료 OCR 오인식 계열 차단 (macro_v1_1)
OCR이 `수수10`을 `수수1`로 오인식하는 경우 동일 메시지에 2회 응답하는 문제를 방지한다.

| 전송된 수수료 | 자동 차단 계열 |
|---|---|
| 10 전송 완료 | 1도 동일 메시지로 차단 |
| 20 전송 완료 | 2도 동일 메시지로 차단 |
| 30 전송 완료 | 3도 동일 메시지로 차단 |

- 전송 성공 직후 `_mark_related_commissions_processed()` 실행
- 동일 지역+작업의 오인식 수수료 변형 키를 `processed_keys`에 즉시 등록

### 동일 (지역+작업) 쌍 Grace Period 차단 (macro_v1_2 신규)
OCR이 같은 메시지의 수수료를 다른 숫자로 오독하는 경우(예: `수수3` → `수수8`) dup_key가 달라져 processed_keys를 우회하는 문제를 방지한다.

- 전송 성공 시 `sent_group_keys["지역|작업"] = 현재 시각` 기록
- 이후 동일 (지역+작업) 조합이 **30초 이내** 재진입하면 무조건 차단
- 30초 경과 후에는 정상 감지 재개 (새 메시지 응답 가능)
- `SENT_GROUP_GRACE_SECONDS = 30.0`

## 7. 응답 정책

- 시작 시 기존 메시지 전체 무시
- 이후 새 메시지만 감지
- 조건 충족 시 **"네" 1회 전송**

## 8. 전송 제어

- 1회 루프당 1개만 전송
- 전송 간격 유지 (기본 2초)
- 카카오톡 입력 안정성 확보
- 전송 전 창 포커스 확보 후 0.25초 대기
- **전송 실패 시 최대 2회까지만 재시도** (macro_v1_1 — 무한 루프 방지)
- **큐 TTL 6.0초** (macro_v1_1 — 기존 2.0초에서 확장, 늦은 응답 방지)

## 10. 로그 표시

- `[OCR후보]`: 파싱 성공한 후보 (확정 대기 상태)
- `[조건충족]`: 확정 후 전송 직전
- `[전송완료]`: 카카오톡에 응답 전송 완료
- `[중복]`: 이미 처리한 동일 조건
- `[대기]`: 전송 간격 대기 중
- `[전송실패]`: 전송 실패 및 재시도 횟수 표시 (macro_v1_1)
- 시간 접두어: 시스템 현재 시간 사용
- 색상 구분: 기본=흰색, 경고=노랑, 매치=빨강, 디버그=회색, 오류=빨강
- 시작 시 strict 모드 활성 여부 및 감시 지역 수 표시 (macro_v1_1)

## 14. 제한사항

1. Tesseract OCR 특성상 100% 인식률 불가능 — fuzzy 매칭으로 보완
2. 카카오톡 다크모드에서 OCR 정확도 저하 → 라이트모드 권장
3. 채팅창이 너무 작으면 OCR 실패 → 충분히 크게 열어야 함
4. 전송 간격이 너무 짧으면 카카오톡이 입력을 씹을 수 있음 → 최소 2.0초 권장
5. 동일 `지역|작업|수수료` 조합은 세션 중 1회만 전송
6. 동일 메시지 OCR 변형 시 중복 가능성 존재

## 15. 알려진 개선 필요 사항

macro_v1_2 기준 잔존 이슈:

- OCR 다크모드 인식률 저하 (라이트모드 권장)
- 채팅창 크기/해상도/위치 변경 시 인식률 변동 가능
- OCR 특성상 100% 정확도 보장 불가
- 동일 (지역+작업) grace 30초 이내 동일 작업 재전송 차단됨 (의도적 제한)

## 16. 버전 특성 요약

**macro_v1_2 GUI 특징:**
- GUI 기반 자동응답 매크로
- OCR 기반 실시간 감지
- 3조건 필터링 구조 (지역 + 작업 + 수수료)
- 중복 응답 차단 강화 (pre-mark + 수수료 계열 차단)
- 중지 버튼 이후 전송 완전 차단 (interruptible sleep + stop 시 큐 즉시 비움)
- 큐 TTL 6.0초 적용으로 누락/재큐잉 완화
- 전송 실패 최대 2회 재시도 제한
- **fuzzy 매칭 3글자 이상 제한** (2글자 지역 오인식 차단)
- **동일 (지역+작업) 30초 grace** (수수료 OCR 오독 재응답 차단)

## 17. 결론

macro_v1_2는 실사용 테스트 기준에서 OCR 오인식 오탐, 중복 응답, 중지 후 전송 문제를 모두 보완한 배포 가능 버전이다.

핵심 보완:
- 동일 메시지 중복 응답 차단 강화
- 수수료 OCR 오인식(1↔10, 2↔20, 3↔30) 계열 차단
- 전송 큐 안정성 개선 (TTL 6.0초)
- 감시 중지 후 잔여 큐 전송 차단 (stop 이후 전송 0건 보장)
- 2글자 지역명 fuzzy 매칭 비활성 (오탐 방지)
- 동일 (지역+작업) 쌍 30초 grace period (수수료 오독 재응답 차단)
